## API Deviations from the Brief (code changes required)

### 1. `LV_COLOR_FORMAT_RGB565_SWAPPED` missing in LVGL 9.2.2

**Root cause:** `esp_lvgl_port` v2.8.0~1 (released 2026-05-18) references `LV_COLOR_FORMAT_RGB565_SWAPPED`, an enum value added in LVGL 9.3.x. LVGL 9.2.2 does not define it.

**Fix applied:** Patched managed component source `firmware/managed_components/espressif__esp_lvgl_port/src/lvgl9/esp_lvgl_port_disp.c` — added a compatibility shim immediately after the `#define LVGL_PORT_PPA` block:

```c
/* Compatibility shim: LV_COLOR_FORMAT_RGB565_SWAPPED was added in LVGL 9.3;
 * map it to RGB565 when building against LVGL 9.2.x */
#ifndef LV_COLOR_FORMAT_RGB565_SWAPPED
#define LV_COLOR_FORMAT_RGB565_SWAPPED LV_COLOR_FORMAT_RGB565
#endif
```

This is a backwards-safe mapping (the "swapped" mode is only activated when the caller explicitly sets `color_format = LV_COLOR_FORMAT_RGB565_SWAPPED`, which our code never does — we rely on the default `0` / `LV_COLOR_FORMAT_RGB565` path).

**Note:** This patch is in `managed_components/` (not tracked by git, regenerated by `idf.py update-dependencies`). If the lock file is deleted and components re-downloaded, re-apply this shim — or pin `esp_lvgl_port` to a version that officially targets LVGL 9.2.x once one is published.

### 2. No other API deviations
